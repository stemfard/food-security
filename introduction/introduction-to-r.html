1 Introduction

Welcome to the foundation of R programming. Before you can perform data analysis, build visualizations, or create models, you must understand how R stores and manipulates information. This session focuses on two fundamental concepts: objects (where R stores data) and variables (the names you give to those objects).

Mastering these basics will make every subsequent step—from data wrangling to statistical modeling—more clear and efficient.

3 What are Objects and Variables?

In R, everything is an object – numbers, text, functions, plots, even the output of statistical models. An object is simply a collection of data stored in memory.

A variable is a name you create that points to an object. Think of variables as labels or handles that let you access and work with data.

# Variable stores a value
x <- 10        # x is a variable holding the numeric object 10
name <- "James" # name is a variable holding the character object "James"

🔹 The Assignment Operator

R offers several ways to assign values to variables. The most common and idiomatic is <-.

Operator Example When to use
<- x <- 5 Preferred – The standard R assignment operator
= x = 5 Allowed but less common; typically used inside function arguments
<<- x <<- 5 Special case – assigns in parent environments (rarely needed)
-> 5 -> x Rightward assignment – valid but unconventional

Best practice:Always use <- for assignment. It signals “I am creating a variable” versus = which R also uses for specifying arguments to functions.

# Good practice
total_revenue <- 150000

# Avoid (though it works)
total_revenue = 150000

# Avoid – confusing to read
150000 -> total_revenue

You can also assign multiple variables in one line:

# Multiple assignments
a <- b <- c <- 10   # a, b, and c all get the value 10

# Or assign from a function
sqrt_x <- sqrt(16)  # sqrt_x becomes 4

4 Naming Variables

🔹 Rules (Must follow – R will error if you break these)

The rules below must be strictly adhered to. Any violation will result in a syntax error.

  1. Start with a letter or a dot (not followed by a number) – name, .hidden, but not 1var or _var

  2. Use only letters, numbers, dot (.), and underscore (_) – No spaces and special characters @, $, %, etc.

  3. No reserved words – if, else, function, TRUE, NULL, NA, for, while, break, next, repeat

# Valid names
age_years <- 1
age.years <- 2
ageYears <- 3
.age_hidden <- 4   # dot at start creates a hidden variable

# Invalid names – these will throw errors
# 1var <- 10       # starts with number
# my-var <- 20     # hyphen not allowed (minus operator)
# my var <- 30     # space not allowed
# if <- 40         # reserved word

🔹 Conventions

These are not enforced by R, but following them makes your code easier for others (and future you) to read.

Convention Example Notes
snake_case total_revenue Most common in R; underscores separate words
camelCase totalRevenue Common in other languages; acceptable in R
period.separated total.revenue Valid but can be confused with S3 method notation
Uppercase for constants MAX_VALUE Signals a value that shouldn’t change

.

Practical tips:

  • Use descriptive, meaningful names – age_in_years not a or x

  • Be consistent within a project – don’t mix total_revenue and total.expenses and totalSpend

  • Avoid overly long names – customer_monthly_average_purchase_amount is precise but tedious

  • Prefix grouped variables – cust_id, cust_name, cust_age

# Good naming examples
customer_age <- 34
is_active <- TRUE
order_total_usd <- 299.99

# Poor naming examples
x <- 34                # meaningless
customer_age_in_years_as_of_today <- 34  # too long
a <- TRUE              # cryptic

5 Getting Help in R

R has excellent built-in documentation. Learning to access it efficiently will save you hours of searching online.

Within RStudio / R console:

Command What it does
?function_name Opens help page for a function (e.g., ?mean)
??keyword Searches for keyword across all help pages
example(function_name) Runs the examples shown in the help page
args(function_name) Shows just the arguments a function takes
vignette("package_name") Opens a longer tutorial for a package
browseVignettes() Lists all available vignettes

.

Common help scenarios:

Task Command Notes
Forgotten how mean() works ?mean Opens help page in viewer pane
Find functions related to correlation ??correlation Searches across all installed packages
See an example of logistic regression example(glm) Runs built-in examples for glm()
Check arguments for plot() args(plot) Shows parameter names and defaults
Long-form guide to dplyr vignette("dplyr") Requires dplyr to be installed

6 Packages

Base R is powerful, but packages extend it enormously. A package is a collection of functions, data, and documentation bundled together.

Why use packages?

  • Add new statistical methods (e.g., lme4 for mixed models)

  • Provide consistent data manipulation tools (e.g., dplyr)

  • Create beautiful graphics (e.g., ggplot2)

  • Connect to external systems (e.g., DBI for databases)

The package workflow:

Task Code Notes
Install a single package install.packages("dplyr") Do once per computer; needs internet connection
Install multiple packages install.packages(c("ggplot2", "tidyr", "readr")) Use c() to combine package names
Install from GitHub remotes::install_github("tidyverse/dplyr") Requires the remotes package first
Load a package library(dplyr) Do once per R session
Check if package is installed require(dplyr) Returns FALSE if not installed (instead of error)
See loaded packages search() Shows all packages currently attached
Update all packages update.packages() Can take a while; answer prompts carefully
See installed packages installed.packages() Returns a large matrix of package metadata
WarningWarning

The code above is for reference only. Do not run install.packages(), update.packages(), or installed.packages() unless you intentionally want to modify your R environment. Loading packages with library() and viewing loaded packages with search() are safe to run.

Key package collections to know:

Collection Purpose
tidyverse Data science – dplyr, ggplot2, tidyr, readr, purrr, stringr
data.table Fast large-data manipulation
shiny Interactive web applications
plotly Interactive graphics
rmarkdown / quarto Reproducible reports

Loading order matters: If you load a package that masks (overrides) a base R function, the package version takes precedence. R will warn you:

# Example warning (for illustration only)
library(dplyr)
Warning: package 'dplyr' was built under R version 4.5.3

Attaching package: 'dplyr'
The following objects are masked from 'package:stats':

    filter, lag
The following objects are masked from 'package:base':

    intersect, setdiff, setequal, union

This means that filter() and lag() are now taken from the dplyr package rather than base R. To explicitly use the base R version, you can call stats::filter().

Similarly, the base R functions intersect(), setdiff(), setequal(), and union() have been masked (overwritten) by functions from dplyr.

CautionCaution

Never use install.packages() inside a script that others will run. It will reinstall every time. Put installation instructions in a separate setup script or README.
