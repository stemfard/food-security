1 What is a Function?

A function in R is a block of reusable code that performs a specific task. Functions help avoid repetition, organize code, and enable modular programming.

Key characteristics:

  • Takes zero or more inputs (arguments)
  • Returns a single output (can be a list, vector, or any object)
  • Has its own local environment

2 Basic Syntax

function_name <- function(argument1, argument2, ...) {
  # Body of the function (Statements to execute)
  return(output)
}

Simple Example: Function to Add two Numbers

Create the function

add_numbers <- function(a, b) {
  result <- a + b
  return(result)
}

Call the function

add_numbers(5, 3)
[1] 8

3 Components of a Function

Component Description
Name Identifies the function (optional for anonymous functions)
Arguments Input parameters (can have default values)
Body Code enclosed in { }
Return value What the function outputs (explicit or implicit)

.

Implicit return

If you don’t use return(), R returns the last evaluated expression:

multiply <- function(x, y) {
  # Even though there is no explicit `return()`,
  # the last evaluated expression is returned
  x * y
}
multiply(4, 5)
[1] 20
NoteExplicit Return

Although not mandatory, using explicit return() improves clarity (good practice in teaching and production code).

3.4.2 Infix Functions

Functions with special names wrapped in % can be used as operators:

`%plus%` <- function(a, b) {
  return(a + b + 100)
}
3 %plus% 5
[1] 108

3.4.3 Nested Functions

A nested function is a function defined inside another function.

Key idea:

  • Defined inside another function
  • Has access to the outer function’s variables (lexical scoping in R)
  • Not necessarily intended as a reusable “helper” outside

3.4.4 Subfunctions

A subfunction is a more informal term. It usually means:

A helper function that supports a main function.

Key idea:

  • Not strictly defined by location
  • Can be nested OR defined separately
  • Emphasizes role, not structure

Key difference between nested functions and subfunctions

Aspect Nested function Subfunction
Definition location Inside another function Inside or outside
Focus Structure (scope-based) Role (helper function)
Scope access Has access to parent environment Depends on definition
Formality More formal term Informal term

3.4.6 Functionals (Functions that Work on Functions)

A functional is a function that takes another function as an argument or returns a function. Functionals are the foundation of elegant, loop-free R code.

3.4.6.1 Why Functionals Matter

Instead of writing loops:

result <- list()
for(i in 1:10) {
  result[[i]] <- mean(my_list[[i]])
}
Error:
! object 'my_list' not found

Use functional

result <- lapply(my_list, mean)  # One line, no loop
Error:
! object 'my_list' not found

Multiple input vectors mapply() map2(), pmap() Handles 2+ arguments

3.6 Best Practices

Below are a few Code Review Standards for Functions

Practice Why It Matters
Function name is a verb and noun calculate_mean not calc_mean
Fewer than 5 arguments Easier to use, less confusion
Less than 30 lines Testable, readable, debuggable
No side effects Only uses inputs, only returns output
Input validation Fails fast with clear message
Documented with Roxygen2 Team can use without asking you
Handles NA values Production data always has gaps
Vectorized (no loops) Works on real dataset sizes
