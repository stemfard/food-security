1 Introduction

R has six atomic data types (also called “atomic vectors” when multiple values are combined). Think of these as the alphabet of the R language — every complex object you’ll ever create is built from these fundamental building blocks. Even sophisticated structures like dates, factors, matrices, and data frames are simply classes constructed on top of these six core data types. We’ll explore each one in Sections 1.1 through 1.6 below.

2 Double

🔹 Definition

A double (or numeric) represents real numbers, including those with decimal values. This is the default numeric type in R.

  • Default numeric type in R
  • 64-bit double precision (approximately 15-16 significant digits)
  • Examples: 3.14, -42.5, 2.9979e8 (scientific notation)

🔹 Syntax

x <- 18.342 # has decimals
typeof(x)
[1] "double"

🔹 Key Notes

  • Can store decimals
  • Higher precision than integers
  • Used in most mathematical computations

🔹 Real-World Examples

  • Temperature in degrees → 36.8
  • Bank balance → 10431.75
  • Height in meters → 1.58
  • The constant pi → 3.141593

🔹 Special Numeric Values

Value Meaning When It Occurs Example
Inf Positive infinity Division by zero 1 / 0
-Inf Negative infinity Division by negative zero -1 / 0
NaN Not a Number Indeterminate form 0 / 0 OR log(-4)
NA Missing value Missing data as.numeric(NA)

3 Integer

🔹 Definition

An integer in R represents whole numbers (no decimals), such as counts or discrete values.

  • Stored internally as 32-bit signed values
  • Must be explicitly declared using L suffix
  • Examples: 42L, -7L, 0L
  • Range: -2147483647 to 2147483647

🔹 Syntax

hh_size <- 8L # note the suffix L
typeof(hh_size)
[1] "integer"

🔹 Key Notes

  • No fractional component
  • Efficient for memory when working with large datasets
  • Default numeric values in R are NOT integers unless specified

🔹 Real-World Examples

  • Number of students in a class → 45L
  • Number of products sold → 1200L
  • Number of meals taken in a day → 3L
  • Number of mosquito nets distributed → 5

🔹 Code Examples

Calculate total calories

calories_breakfast <- 450L
calories_lunch <- 680L

total_calories <- calories_breakfast + calories_lunch
print(total_calories)
[1] 1130

Calculate portfolio

tesla_shares <- 50L
apple_shares <- 100L

total_shares <- tesla_shares + apple_shares
print(total_shares)
[1] 150

4 Character

🔹 Definition

The character data type in R is used to store textual data (strings), such as names, labels, codes, or any sequence of letters, numbers, and symbols enclosed in quotes.

🔹 Syntax

name <- "STEM Research"
typeof(name)
[1] "character"

🔹 Key Notes

  • Must be enclosed in single (' ') or double (" ") quotes
  • Numbers stored as characters are not numeric (e.g., "123" ≠ 123)
  • Commonly used for categorical labels, identifiers, and descriptive data

🔹 Real-World Examples

Category Example Why Character?
Patient name "John Doe" Names contain letters, spaces, and sometimes hyphens or apostrophes
Email address "user@example.com" Contains special characters (@, ., etc.) that cannot be numeric
Phone number "+1-555-123-4567" Includes country codes, dashes, parentheses, or leading zeros
Product SKU "XL-BLUE-042" Mix of letters, hyphens, and numbers (where the number is an identifier, not a value)
Postal/ZIP code "90210" or "SW1A 1AA" Can have leading zeros or letters; should not be used in arithmetic
Country code "US", "JP", "DE" Two-letter codes are categorical labels, not mathematical values
DNA sequence "ATCGATCG" String of nucleotide letters representing genetic information
ISBN (book ID) "0-306-40615-2" Contains hyphens and a checksum character that may be “X”
Yes/No response "Yes", "No" Categorical text response from a survey

🔹 Code Examples

country <- "Kenya"
code <- "A123"
number_as_text <- "100"

5 Logical

🔹 Definition

A logical (or boolean) represents binary truth values: either TRUE or FALSE. Logical values are fundamental for conditional operations, filtering data, and control flow in R.

  • Two possible values: TRUE (1) and FALSE (0)
  • Can be abbreviated as T and F (not recommended for production code)
  • Result of comparison and logical operators
  • Examples: TRUE, FALSE

🔹 Syntax

is_adult <- TRUE
typeof(is_adult)
[1] "logical"

🔹 Key Notes

  • Used in conditional statements (if, else, while)
  • Automatically coerces to integer in arithmetic: TRUE → 1, FALSE → 0
  • Results from comparison operators: >, <, ==, !=, >=, <=
  • Can be combined with logical operators: & (AND), | (OR), ! (NOT)

🔹 Real-World Examples

  • Is the patient diabetic? → TRUE
  • Has the loan been approved? → FALSE
  • Is the temperature above 37°C? → TRUE
  • Did the student pass the exam? → TRUE
  • Is the baby underweight? → FALSE

5.1 Conditional Logic

Conditional logic is the foundation of decision-making in programming, allowing R to evaluate conditions and execute different actions based on whether expressions are TRUE or FALSE. It is essential for building dynamic, intelligent, and rule-based systems.

5.1.1 Logical Operators

Logical operators in R enable the combination and evaluation of multiple conditions simultaneously. They form the backbone of complex decision-making by allowing expressions to be joined using AND, OR, and NOT logic.

5.1.2 Relational (Comparison) Operators

Relational operators are used to compare values and determine their relationship, such as equality, inequality, or ordering. These comparisons produce logical outcomes (TRUE or FALSE) that drive conditional execution in R.

5.1.3 Membership and Other Special Operators

Membership and special operators in R are used to test whether elements belong to a set, sequence, or structure, and to perform specialized operations that go beyond basic arithmetic and comparison. They are particularly useful in data filtering and pattern matching.

5.2 Operator Precedence

Operator precedence defines the hierarchy that R follows when evaluating expressions with multiple operators. Understanding this order is crucial to ensuring expressions are interpreted correctly and produce expected results without ambiguity.

ImportantFloating-Point Comparisons

In R, decimal numbers are stored using binary approximations, which can lead to small precision errors when comparing values.

is_equal <- 0.1 + 0.2 == 0.3
is_equal
[1] FALSE

This may return FALSE because of floating-point representation error, even though mathematically the values are equal.

To safely compare decimal numbers in R, use all.equal():

is_equal <- all.equal(0.1 + 0.2, 0.3)
is_equal
[1] TRUE

6 Complex

🔹 Definition

A complex number in R represents numbers with both real and imaginary parts, written as a + bi where i is the square root of -1.

  • Written as real + imaginary i
  • Stored internally as pairs of double-precision numbers
  • Imaginary unit denoted by i or j suffix
  • Examples: 3 + 2i, -4i, 1 + 0i
  • Functions: Re(), Im(), Mod(), Arg()
  • Applications: Signal processing, quantum mechanics, engineering calculations

🔹 Syntax

z <- 3 + 4i # real part = 3, imaginary part = 4
typeof(z)
[1] "complex"

🔹 Key Notes

  • Consists of real part and imaginary part
  • Supports arithmetic operations (+, -, *, /)
  • Can extract real and imaginary components using Re() and Im()
  • Useful for Fourier transforms, AC circuit analysis, and quantum mechanics

🔹 Real-World Examples

  • AC circuit impedance → 50 + 30i ohms
  • Quantum state amplitude → 0.6 + 0.8i
  • Signal frequency component → -2 + 5i
  • Complex root of polynomial → -1 + 2i

7 Raw

🔹 Definition

A raw vector in R represents bytes (0-255) of binary data, such as file contents or encrypted information.

Raw data type is rarely used in standard statistical analysis. This section provides only a brief introduction to the basics.

  • Stored internally as a sequence of bytes (0-255)
  • Must be created using as.raw() or charToRaw()
  • Examples: as.raw(65), charToRaw("Hello")
  • Range: 00 to ff (hexadecimal) or 0 to 255 (decimal)

🔹 Syntax

byte <- as.raw(65) # creates raw byte with value 65 (ASCII 'A')
typeof(byte)
[1] "raw"

🔹 Key Notes

  • Stores binary data exactly as bytes
  • Useful for file I/O, network communication, and cryptography
  • Cannot perform arithmetic operations directly
  • Rarely needed in typical statistical workflows

🔹 Real-World Examples

  • Reading binary file contents → as.raw(c(0x48, 0x69))
  • Text to bytes for encryption → charToRaw("Secret")
  • Image pixel data → readBin("image.jpg", "raw", n = 100)
  • Network packet payload → raw byte sequences

8 Functions for Inspecting Data Types

When working with data in R, you’ll often need to verify what type of data you’re dealing with before performing operations. R provides several functions for checking data types, each serving a slightly different purpose. typeof() reveals the internal storage type, class() shows the object-oriented class, and the family of is.type() functions return simple TRUE/FALSE checks.

Function Description Example Result
typeof(x) Returns the internal storage type of x typeof(42L) "integer"
class(x) Returns the class attribute of x class(3.14) "numeric"
mode(x) Returns the R storage mode of x mode(TRUE) "logical"
str(x) Displays the structure of x str("hello") chr "hello"
is.logical(x) Checks if x is logical is.logical(FALSE) TRUE
is.integer(x) Checks if x is integer is.integer(42) FALSE
is.double(x) Checks if x is double is.double(3.14) TRUE
is.numeric(x) Checks if x is numeric (integer or double) is.numeric(42L) TRUE
is.character(x) Checks if x is character is.character("R") TRUE
is.complex(x) Checks if x is complex is.complex(3+4i) TRUE
is.raw(x) Checks if x is raw is.raw(as.raw(65)) TRUE
is.atomic(x) Checks if x is an atomic vector is.atomic(3.14) TRUE
is.vector(x) Checks if x is a vector is.vector(42L) TRUE

9 Type Coercion

Type coercion converts data from one type to another. R performs implicit coercion automatically in some contexts, but explicit coercion gives you control. Understanding coercion prevents silent errors and unexpected results.

Function Description Example Result
as.logical(x) Converts x to logical as.logical(42) TRUE
as.integer(x) Converts x to integer as.integer(3.14) 3L
as.double(x) Converts x to double as.double(42L) 42
as.numeric(x) Converts x to numeric (alias for as.double) as.numeric(TRUE) 1
as.character(x) Converts x to character as.character(3.14) "3.14"
as.complex(x) Converts x to complex as.complex(3) 3+0i
as.raw(x) Converts x to raw as.raw(65) 41
