1 Introduction

Conditional logic is the foundation of decision-making in programming, allowing R to evaluate conditions and execute different actions based on whether expressions are TRUE or FALSE. It is essential for building dynamic, intelligent, and rule-based systems.

2 Logical Operators

Logical operators in R enable the combination and evaluation of multiple conditions simultaneously. They form the backbone of complex decision-making by allowing expressions to be joined using AND, OR, and NOT logic.

Operator Meaning Example Result
& AND (vectorized) c(TRUE, TRUE) & c(TRUE, FALSE) TRUE FALSE
| OR (vectorized) c(TRUE, FALSE) | c(FALSE, FALSE) TRUE FALSE
! NOT !TRUE FALSE
&& AND (scalar, short-circuit) TRUE && FALSE FALSE
|| OR (scalar, short-circuit) TRUE || FALSE TRUE
xor() Exclusive OR (exactly one is TRUE) xor(TRUE, TRUE) FALSE
isTRUE() Checks if expression is TRUE isTRUE(5 > 3) TRUE
isFALSE() Checks if expression is FALSE isFALSE(5 > 3) FALSE

3 Relational (Comparison) Operators

Relational operators are used to compare values and determine their relationship, such as equality, inequality, or ordering. These comparisons produce logical outcomes (TRUE or FALSE) that drive conditional execution in R.

Operator Meaning Example Result
== Equal to 5 == 5 TRUE
!= Not equal to 5 != 3 TRUE
> Greater than 5 > 3 TRUE
< Less than 5 < 3 FALSE
>= Greater than or equal to 5 >= 5 TRUE
<= Less than or equal to 3 <= 5 TRUE

4 Membership and Other Special Operators

Membership and special operators in R are used to test whether elements belong to a set, sequence, or structure, and to perform specialized operations that go beyond basic arithmetic and comparison. They are particularly useful in data filtering and pattern matching.

Operator Meaning Example Result
%in% Value matching (membership) "apple" %in% c("apple", "banana") TRUE
%in% Multiple values membership c(2, 5) %in% c(1, 2, 3, 4) TRUE FALSE
identical() Tests exact equality of objects identical(1L, 1) FALSE
all.equal() Tests near equality (numeric tolerance) all.equal(0.1 + 0.2, 0.3) TRUE
any() At least one element is TRUE any(c(FALSE, FALSE, TRUE)) TRUE
all() All elements are TRUE all(c(TRUE, TRUE, FALSE)) FALSE

5 Operator Precedence

Operator precedence defines the hierarchy that R follows when evaluating expressions with multiple operators. Understanding this order is crucial to ensuring expressions are interpreted correctly and produce expected results without ambiguity.

Precedence Operators Description
1 () Parentheses
2 ! Logical NOT
3 : Sequence operator
4 *, / Multiplication, Division
5 +, - Addition, Subtraction
6 >, <, >=, <= Relational operators
7 ==, != Equality operators
8 & Logical AND (vectorized)
9 && Logical AND (short-circuit)
10 | Logical OR (vectorized)
11 || Logical OR (short-circuit)
12 = Assignment (left to right)
13 <- Assignment (right to left)
